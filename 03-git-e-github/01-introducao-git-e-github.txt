O Git é um sistema de controle de versionamento de código distribuído e amplamente adotado. O Git nasceu e foi tomando espaço dos outros sistemas de controle. 
Seu criador principal é o mesmo que o do Linux: Linus Torvalds.
O Github tem sim muita relação com o Git. 
GitHub é uma plataforma para gerenciar seu código e criar um ambiente de colaboração entre devs, utilizando o Git como sistema de controle.

1 - Controle de versão
2 - Armazenamento em núvem
3 - Trabalho em equipe
4 - Melhorar seu código
5 - Reconhecimento

Navegação básica no terminal e instalação

GUI x CLI
Grafic User Interface x Comand Line Interface
Git não possui interface gráfica e por isso é preciso usá-lo via linha de comando.

Comandos básicos

Caminha para uma pasta específica no sistema operacional : cd
Listar diretórios: dir
Navegar diretamente até a raiz (C:\): cd / 
Retornar um nível na pasta: cd ..
Navegar até uma pasta: cd NomeDaPasta
Limpar o terminal (clear screen): cls
Auto completar (como nome de pastas): TAB
Criar uma pasta (make dir): mkdir NomeDaPasta
Printar de volta na tela o que for digitado: echo Hello
Criar um arquivo em um diretório: echo hello > hello.txt
Deletar todo conteúdo de um diretório: del nomeDoDiretorio
Deletar um diretório (remove directory): rmdir nomeDoDiretorio /s /q
